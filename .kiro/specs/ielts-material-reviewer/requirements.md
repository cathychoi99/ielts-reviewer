# 需求文档

## 简介

雅思备考英语材料复习网站应用（IELTS Material Reviewer）是一个面向雅思考生的单用户 Web 应用。用户可以导入英语学习材料（文章、播客文稿、Vlog 脚本等），系统通过 AI 自动提取生词、词组搭配和优质句子表达，并根据用户的英语水平和雅思考试重点进行优先级标注。用户可以管理摘录、标记掌握状态，并通过卡片式复习模式进行回顾。

技术栈：React + TypeScript + Tailwind CSS 前端，Node.js (Express) + SQLite 后端，通过 OpenAI SDK 兼容接口对接 DeepSeek 或其他 AI 服务。

## 术语表

- **材料（Material）**：用户导入的一篇英语学习内容，包含标题、来源标签、原文文本
- **摘录（Extraction）**：AI 从材料中提取的学习条目，分为词汇、词组、句子三种类型
- **词汇摘录（Vocabulary_Extraction）**：从材料中提取的生词或高频雅思词汇，包含释义、词性、例句
- **词组摘录（Collocation_Extraction）**：从材料中提取的词组搭配（collocation），包含释义和例句
- **句子摘录（Sentence_Extraction）**：从材料中提取的值得学习的句子表达，包含分析说明和适用场景
- **来源标签（Source_Tag）**：材料的来源分类，取值为 Vlog、文章、播客、其他
- **英语水平（Band_Level）**：用户自评的雅思分数等级，取值为 5.0、5.5、6.0、6.5、7+，影响 AI 摘录的难度阈值
- **掌握状态（Mastery_Status）**：摘录的学习状态，取值为"未掌握"或"已掌握"
- **复习模式（Review_Mode）**：卡片式复习功能，随机展示摘录并支持翻转查看释义
- **AI_Service**：通过 OpenAI SDK 兼容接口调用的大语言模型服务，用于分析材料并提取摘录
- **解析器（Parser）**：负责处理 AI 返回结果并转换为结构化摘录数据的模块
- **材料库（Material_Library）**：首页展示的所有已导入材料的列表视图
- **摘录本（Extraction_Book）**：跨材料汇总所有摘录的集中视图

## 需求

### 需求 1：材料输入

**用户故事：** 作为雅思考生，我想导入英语学习材料，以便系统帮我分析和提取学习要点。

#### 验收标准

1. WHEN 用户在添加材料页面粘贴纯文本，THE 材料输入模块 SHALL 接受该文本作为材料内容
2. WHEN 用户在添加材料页面上传 .txt 或 .md 格式的文件，THE 材料输入模块 SHALL 读取文件内容并作为材料内容
3. IF 用户上传的文件格式不是 .txt 或 .md，THEN THE 材料输入模块 SHALL 显示错误提示"仅支持 .txt 和 .md 格式的文件"
4. THE 材料输入模块 SHALL 要求用户为每篇材料填写标题
5. THE 材料输入模块 SHALL 要求用户为每篇材料选择一个来源标签（Vlog / 文章 / 播客 / 其他）
6. WHEN 用户提交材料，THE 材料输入模块 SHALL 将材料标题、来源标签和原文文本持久化存储到数据库
7. IF 用户提交材料时标题为空或内容为空，THEN THE 材料输入模块 SHALL 显示对应的校验错误提示并阻止提交

### 需求 2：AI 智能解析与摘录提取

**用户故事：** 作为雅思考生，我想让系统自动分析我的学习材料并提取关键学习要点，以便我高效地积累词汇和表达。

#### 验收标准

1. WHEN 用户提交材料后点击"开始解析"按钮，THE AI_Service SHALL 分析材料内容并提取词汇摘录、词组摘录和句子摘录
2. THE AI_Service SHALL 为每个词汇摘录提供单词、释义、词性和一个例句
3. THE AI_Service SHALL 为每个词组摘录提供词组文本、释义和一个例句
4. THE AI_Service SHALL 为每个句子摘录提供原句、分析说明（为什么值得学习）和适用场景（写作/口语等）
5. THE AI_Service SHALL 根据雅思考试重点（学术词汇、写作高分表达、口语高分表达）为每个摘录标注优先级（高/中/低）
6. WHEN 用户已设置英语水平（Band_Level），THE AI_Service SHALL 根据该水平调整摘录的难度阈值，过滤掉低于用户水平的基础内容
7. WHILE AI 解析正在进行中，THE 材料详情页 SHALL 显示加载状态指示器
8. WHEN AI 解析完成，THE 解析器 SHALL 将 AI 返回的结果转换为结构化的摘录数据并存储到数据库
9. IF AI_Service 调用失败或返回无效数据，THEN THE 解析器 SHALL 显示错误提示并允许用户重新触发解析
10. THE 解析器 SHALL 将 AI 返回的 JSON 响应解析为结构化的摘录对象
11. THE 解析器 SHALL 将结构化的摘录对象格式化为可存储的 JSON 字符串
12. FOR ALL 有效的摘录对象，解析后格式化再解析 SHALL 产生等价的摘录对象（往返一致性）

### 需求 3：摘录管理

**用户故事：** 作为雅思考生，我想管理和浏览所有提取的学习要点，以便有针对性地复习。

#### 验收标准

1. THE 材料详情页 SHALL 按类型（词汇/词组/句子）分组展示该材料的所有摘录
2. WHEN 用户在材料详情页选择某个类型筛选器，THE 材料详情页 SHALL 仅显示该类型的摘录
3. WHEN 用户点击摘录卡片上的"标记已掌握"按钮，THE 摘录管理模块 SHALL 将该摘录的掌握状态更新为"已掌握"
4. WHEN 用户点击已掌握摘录上的"取消掌握"按钮，THE 摘录管理模块 SHALL 将该摘录的掌握状态恢复为"未掌握"
5. THE 摘录本页面 SHALL 汇总展示所有材料的摘录
6. WHEN 用户在摘录本页面选择筛选条件（类型、掌握状态、来源标签），THE 摘录本页面 SHALL 根据所选条件过滤摘录列表
7. THE 摘录卡片 SHALL 根据摘录类型展示对应的信息字段（词汇显示释义和词性，词组显示释义，句子显示分析说明和适用场景）

### 需求 4：卡片式复习模式

**用户故事：** 作为雅思考生，我想通过卡片翻转的方式复习摘录，以便加深记忆。

#### 验收标准

1. WHEN 用户进入复习模式页面，THE 复习模式 SHALL 允许用户选择复习范围（全部摘录或指定材料的摘录）和摘录类型（词汇/词组/句子/全部）
2. WHEN 用户开始复习，THE 复习模式 SHALL 随机排列所选范围内未掌握的摘录并逐张展示卡片
3. THE 复习卡片正面 SHALL 显示摘录的核心内容（词汇显示单词、词组显示词组文本、句子显示原句）
4. WHEN 用户点击复习卡片，THE 复习卡片 SHALL 翻转显示详细信息（释义、词性、例句、分析说明等）
5. THE 复习模式 SHALL 显示当前复习进度（如"第 3/20 张"）
6. WHEN 用户在复习模式中点击"已掌握"按钮，THE 复习模式 SHALL 将该摘录标记为已掌握并展示下一张卡片
7. WHEN 所有卡片复习完毕，THE 复习模式 SHALL 显示复习完成的总结信息（复习总数、本次标记已掌握数量）

### 需求 5：材料管理

**用户故事：** 作为雅思考生，我想查看和管理所有导入的材料，以便回顾学习历史。

#### 验收标准

1. THE 材料库页面 SHALL 以卡片列表形式展示所有已导入的材料，每张卡片显示标题、来源标签、摘录统计数量和添加时间
2. THE 材料库页面 SHALL 按添加时间倒序排列材料（最新的在前）
3. WHEN 用户点击材料卡片，THE 材料库页面 SHALL 导航到该材料的详情页
4. THE 材料详情页 SHALL 提供两个 Tab：原文 Tab 和摘录 Tab
5. WHEN 用户查看原文 Tab，THE 材料详情页 SHALL 显示材料的完整原文，并对已提取的词汇和词组进行高亮标注
6. WHEN 用户查看摘录 Tab，THE 材料详情页 SHALL 显示该材料的所有摘录卡片列表

### 需求 6：用户设置

**用户故事：** 作为雅思考生，我想配置我的英语水平和 AI 服务参数，以便系统提供适合我水平的学习内容。

#### 验收标准

1. THE 设置页面 SHALL 提供英语水平选择器，选项为 Band 5.0、Band 5.5、Band 6.0、Band 6.5、Band 7+
2. WHEN 用户选择英语水平并保存，THE 设置模块 SHALL 持久化存储该设置
3. THE 设置页面 SHALL 提供 AI API Key 输入框和 API Base URL 输入框
4. THE 设置页面 SHALL 将 API Base URL 默认预设为 DeepSeek 的 API 地址
5. WHEN 用户保存 AI 配置，THE 设置模块 SHALL 持久化存储 API Key 和 API Base URL
6. IF 用户未配置 AI API Key 就尝试解析材料，THEN THE AI_Service SHALL 显示提示"请先在设置中配置 AI API Key"

### 需求 7：响应式设计

**用户故事：** 作为雅思考生，我想在手机和电脑上都能方便地使用这个应用，以便随时随地复习。

#### 验收标准

1. THE 应用 SHALL 使用 Tailwind CSS 响应式工具类实现自适应布局
2. WHILE 屏幕宽度小于 768px，THE 应用 SHALL 以移动端优化的单列布局展示内容
3. WHILE 屏幕宽度大于等于 768px，THE 应用 SHALL 以桌面端优化的多列布局展示内容
4. THE 复习卡片翻转交互 SHALL 在触屏设备上支持点击触发

### 需求 8：页面导航

**用户故事：** 作为雅思考生，我想在各个功能页面之间方便地切换，以便流畅地使用应用。

#### 验收标准

1. THE 应用 SHALL 提供持久化的导航栏，包含以下入口：材料库（/）、摘录本（/extractions）、复习模式（/review）、设置（/settings）
2. WHEN 用户点击导航栏中的入口，THE 应用 SHALL 使用客户端路由导航到对应页面，无需整页刷新
3. THE 导航栏 SHALL 高亮显示当前所在页面的对应入口
4. WHEN 用户在材料库页面点击"添加新材料"按钮，THE 应用 SHALL 导航到添加材料页面（/materials/new）
